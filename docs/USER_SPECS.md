## 1. Регистрация и вход

### Регистрация

Откройте приложение в браузере (по умолчанию `https://localhost`). Нажмите **«Зарегистрироваться»** и заполните форму:

- **Email** — должен быть уникальным
- **Имя пользователя** — от 3 до 64 символов, уникальное
- **Пароль** — без ограничений по длине, рекомендуется минимум 12 символов

После регистрации система сразу выполнит вход.

### Вход

На форме входа введите email и пароль. Если включена двухфакторная аутентификация — появится второй шаг с запросом TOTP-кода из приложения-аутентификатора.

Также можно войти через **GitHub** — кнопка на форме входа. При первом входе через OAuth создаётся новый аккаунт, при повторных — используется существующий.

### Автообновление токена

Сессия активна 30 дней. Access-токен обновляется автоматически в фоне — вы не должны замечать разрывов при работе.

## 2. Настройка профиля

Перейдите в профиль через иконку в правом верхнем углу или по пути `/profile`.

### Аватар

Нажмите на круглую область с аватаром — откроется диалог выбора файла. Поддерживаются форматы JPEG, PNG, WebP, GIF. Максимальный размер — 5 МБ. После загрузки аватар обновится на всех страницах.

### Имя и email

В карточке «Личные данные» можно изменить имя пользователя и email. Email после изменения потребует повторного входа.

### Смена пароля

Карточка «Смена пароля» — введите текущий пароль и новый дважды. Минимальных требований к сложности нет, но рекомендуется не менее 12 символов.

## 3. Двухфакторная аутентификация

Двухфакторная аутентификация (2FA) работает через любое TOTP-совместимое приложение: Google Authenticator, Authy, Microsoft Authenticator, 1Password и другие.

### Включение 2FA

1. Откройте профиль → раздел «Двухфакторная аутентификация»
2. Нажмите **«Включить 2FA»**
3. Отсканируйте QR-код в приложении-аутентификаторе или введите секретный ключ вручную
4. Введите 6-значный код из приложения для подтверждения
5. Нажмите **«Активировать»**

После этого при каждом входе система будет запрашивать код из приложения.

### Отключение 2FA

В том же разделе — кнопка **«Отключить 2FA»**. Потребуется ввести текущий TOTP-код.

> Если потеряли доступ к приложению-аутентификатору — обратитесь к администратору для сброса 2FA через прямой доступ к базе данных.

## 4. Создание скана

Перейдите на **Дашборд** (`/dashboard`) и нажмите **«Новое сканирование»**.

### Основные параметры

**URL цели** — полный адрес сайта вместе со схемой: `https://example.com`. Сканер останется в пределах этого домена, на другие домены не уйдёт.

**Глубина обхода** (1–10, по умолчанию 3) — максимальное количество переходов по ссылкам от стартовой страницы. Глубина 1 проверит только стартовую страницу и ссылки с неё. Для большинства сайтов хватает 3–4.

**Таймаут** (60–86400 секунд, по умолчанию 3600) — общее время на весь скан. Краулер использует 60% от таймаута, атака — оставшееся.

### Исключённые пути

Пути, которые сканер не будет посещать и атаковать. Один путь на строку:

```
/logout
/admin/dangerous-action
/vulnerabilities/csrf
```

Это важно для защиты самого приложения: например, форма смены пароля может быть уязвима к CSRF, но если сканер отправит туда нагрузку — сменит пароль и потеряет доступ. Добавьте такие пути в исключения.

### Аутентификация

Если сканируемый сайт требует входа — настройте один из режимов:

**Без аутентификации** — сканер работает как анонимный пользователь.

**Cookie** — вставьте строку cookie из браузера (DevTools → Application → Cookies → скопировать как строку). Например: `session=abc123; token=xyz`.

**HTTP Basic Auth** — введите логин и пароль. Сканер будет отправлять заголовок `Authorization: Basic ...` с каждым запросом.

**Bearer Token** — введите JWT или другой Bearer-токен. Отправляется как `Authorization: Bearer <токен>`.

**Form Login** — сканер сам выполнит вход через форму:
- Укажите URL страницы входа (например `https://example.com/login`)
- Имена полей формы (обычно `username` и `password`)
- Логин и пароль

При form login сканер сначала логинится, получает сессионные cookie и уже с ними проводит атаку.

### Запуск

Нажмите **«Запустить сканирование»**. Вас автоматически перенаправит на страницу скана.

## 5. Мониторинг скана

На странице скана отображается:

- **Статус** — текущее состояние (ожидание, выполняется, завершён и т.д.)
- **Статистика** — количество найденных уязвимостей, обойдённых страниц, форм и JS-маршрутов
- **Уязвимости** — таблица находок в реальном времени
- **Лог** — подробный журнал работы сканера

### Управление

Пока скан выполняется, доступны кнопки:

**Пауза** — останавливает текущую атаку. Скан переходит в состояние «Приостановлен». Можно возобновить позже.

**Продолжить** — ставит приостановленный скан обратно в очередь. Атака начнётся заново от последней точки (не с нуля, но без сохранения промежуточного состояния — краулер перезапустится).

**Остановить** — отменяет скан окончательно. Действие необратимо. Все найденные до этого уязвимости сохраняются.

### Лог

Нажмите на **«Лог скана»** чтобы развернуть журнал. Строки добавляются в реальном времени. В логе видно:

- Какие страницы посещает краулер и с какой глубиной
- Сколько форм и JS-маршрутов найдено
- Какие нагрузки тестируются и на каких параметрах
- Что именно нашёл анализатор (`⚠ НАЙДЕНО XSS MEDIUM ...`)
- Ошибки (если что-то пошло не так)

## 6. Просмотр отчёта

После завершения скана (статус «Завершён» или «Остановлен») кнопка **«Отчёт»** станет активной.

### Структура отчёта

**Заголовок** — цель, ID скана, время начала и окончания.

**Статистика** — общее число уязвимостей, количество обойдённых страниц и форм.

**По критичности** — цветовые плашки: Critical (фиолетовый), High (красный), Medium (оранжевый), Low (синий), Info (серый).

**По типу уязвимости** — SQL-инъекция, XSS, SSRF и т.д.

**Таблица уязвимостей** — каждая строка кликабельна. При клике разворачивается панель с подробностями:

- **Payload** — нагрузка, которая сработала
- **reflected_payload** (для XSS) — что именно отразилось в ответе
- **signature** (для SQLi) — найденная сигнатура в теле ответа
- **location** (для Open Redirect) — куда ведёт редирект
- **status** — HTTP-код ответа

### Скачать отчёт

Кнопка **«⬇ JSON»** скачивает полный отчёт как JSON-файл. В нём те же данные что на странице плюс полные URL всех посещённых страниц.

## 7. Управление API-токенами

API-токены позволяют использовать функции сканера без браузера — через curl, Postman или скрипты.

### Создать токен

Профиль → раздел «API-токены» → **«Создать токен»** → введите название (например «postman-tests»).

Токен показывается **один раз** сразу после создания — скопируйте его, потом увидеть будет невозможно.

### Использование

```bash
curl -H "X-API-Key: dast_ваш_токен_здесь" https://localhost/api/scans
```

Токен можно передавать в заголовке `X-API-Key` вместо JWT. Права токена совпадают с правами создавшего его пользователя.

### Отозвать токен

Кнопка «Отозвать» рядом с токеном в списке. После отзыва токен перестаёт работать немедленно.

## 8. Панель администратора

Доступна только пользователям с ролью `admin`. Перейти: `/admin` или через меню навигации.

### Управление пользователями

Таблица всех пользователей системы. Для каждого можно:

- Сменить роль (`user` / `admin`)
- Заблокировать аккаунт (флаг `is_active = false`)

### Управление сессиями

Просмотр активных сессий любого пользователя — видно IP, user-agent, время создания и истечения. Можно принудительно завершить любую сессию.

### Управление API-токенами

Список всех токенов в системе с указанием владельца и времени последнего использования. Можно отозвать любой токен.
