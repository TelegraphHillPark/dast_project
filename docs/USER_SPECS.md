# Руководство пользователя — DAST Analyzer

## 1. Общие сведения

Настоящее руководство предназначено для пользователей системы DAST Analyzer и содержит описание порядка работы с графическим веб-интерфейсом системы.

Система DAST Analyzer предоставляет следующие основные функции:
- регистрация и аутентификация пользователей;
- запуск и управление задачами динамического сканирования веб-ресурсов;
- просмотр отчётов об обнаруженных уязвимостях;
- управление пользовательскими словарями пейлоадов;
- администрирование учётных записей (для пользователей с ролью администратора).

---

## 2. Требования к рабочему месту

Для работы с системой требуется веб-браузер одной из следующих версий или более поздних: Google Chrome 90, Mozilla Firefox 88, Apple Safari 14, Microsoft Edge 90. Специализированное программное обеспечение на рабочем месте пользователя не требуется.

---

## 3. Аутентификация в системе

### 3.1 Регистрация учётной записи

Для регистрации новой учётной записи необходимо:

1. Открыть стартовую страницу приложения в браузере.
2. Нажать кнопку **«Создать аккаунт»**.
3. Заполнить регистрационную форму:
   - **Email** — адрес электронной почты (используется в качестве логина);
   - **Имя пользователя** — уникальный идентификатор, допустимы латинские буквы, цифры и символ подчёркивания, длина от 3 до 64 символов;
   - **Пароль** — не менее 8 символов, должен содержать буквы и цифры.
4. Нажать кнопку **«Зарегистрироваться»**.

При успешной регистрации выполняется автоматическое перенаправление на главную страницу системы.

### 3.2 Вход в систему по логину и паролю

Для входа в систему необходимо:

1. Перейти на страницу входа.
2. Ввести адрес электронной почты и пароль.
3. Нажать кнопку **«Войти»**.

### 3.3 Вход через GitHub OAuth

Для аутентификации с использованием учётной записи GitHub необходимо:

1. На странице входа нажать кнопку **«Войти через GitHub»**.
2. В открывшемся окне GitHub подтвердить предоставление доступа приложению.
3. После успешной авторизации браузер выполнит перенаправление на главную страницу системы.

### 3.4 Двухфакторная аутентификация (2FA)

При включённой двухфакторной аутентификации после ввода логина и пароля система запросит одноразовый код, генерируемый приложением-аутентификатором (Google Authenticator, Authy или аналогичным). Код действителен в течение 30 секунд.

Порядок включения 2FA описан в разделе 7.2 настоящего руководства.

---

## 4. Главная страница (дашборд)

После входа в систему отображается главная страница, содержащая:
- список ранее запущенных сканирований с указанием статуса и даты;
- кнопку создания нового сканирования.

Нажатие на строку сканирования в таблице открывает страницу с подробными сведениями о нём.

---

## 5. Создание и управление сканированием

### 5.1 Создание нового сканирования

Для запуска нового сканирования необходимо:

1. На главной странице нажать кнопку **«Новое сканирование»**.
2. В открывшейся форме указать:
   - **URL цели** — полный адрес сканируемого ресурса, включая схему (`http://` или `https://`);
   - **Словари** (необязательно) — выбрать один или несколько пользовательских словарей пейлоадов из числа предварительно загруженных.
3. Нажать кнопку **«Запустить»**.

> **Предупреждение.** Сканирование допускается проводить исключительно в отношении ресурсов, на тестирование которых получено явное разрешение владельца. Несанкционированное сканирование сторонних ресурсов является нарушением законодательства.

### 5.2 Мониторинг хода сканирования

На странице запущенного сканирования отображается текущий статус и журнал событий. Сканирование проходит следующие фазы:

| Фаза | Описание |
|---|---|
| Краулинг | Автоматический обход страниц, форм и параметров целевого ресурса |
| Анализ заголовков | Проверка HTTP-заголовков ответа на соответствие требованиям безопасности |
| Атака | Инъекция тестовых пейлоадов, анализ реакции сервера |

### 5.3 Управление активным сканированием

В процессе выполнения сканирования доступны следующие действия:

- **Пауза** — временная приостановка сканирования с сохранением прогресса. Возобновление выполняется кнопкой **«Возобновить»**.
- **Отмена** — полная остановка сканирования. Результаты, полученные до момента отмены, сохраняются и доступны в отчёте.

## 6. Просмотр отчётов

### 6.1 Таблица уязвимостей

По завершении сканирования формируется отчёт в виде таблицы, содержащей следующие поля:

| Поле | Описание |
|---|---|
| Тип | Категория уязвимости согласно классификации (SQLi, XSS, SSRF и др.) |
| CWE | Идентификатор уязвимости по каталогу MITRE CWE (является гиперссылкой) |
| URL | Адрес, на котором обнаружена уязвимость |
| Параметр | Уязвимый параметр запроса или поле формы |
| Критичность | Уровень опасности: Critical, High, Medium, Low, Info |
| HTTP-статус | Код ответа сервера при срабатывании |

### 6.2 Детальное описание уязвимости

Нажатие на строку таблицы раскрывает панель с детальными сведениями:

- **curl-команда** — готовая команда командной строки для воспроизведения уязвимости. Кнопка «Скопировать» помещает команду в буфер обмена.
- **Пейлоад** — тестовая нагрузка, спровоцировавшая обнаружение уязвимости.
- **Фрагмент ответа** — извлечённый фрагмент тела HTTP-ответа.
- **OWASP** — идентификатор категории по методологии OWASP Top 10 2021.
- **Рекомендация** — описание меры устранения уязвимости.

### 6.3 Фильтрация результатов

Над таблицей расположены кнопки фильтрации по уровню критичности. При нажатии кнопки отображаются уязвимости только соответствующего уровня. Повторное нажатие на активный фильтр снимает его.

## 7. Управление профилем

Раздел «Профиль» доступен через навигационное меню и содержит следующие функции.

### 7.1 Изменение учётных данных

- **Имя пользователя** — заполнить поле нового имени пользователя и нажать **«Сохранить»**.
- **Пароль** — ввести текущий пароль, новый пароль и его подтверждение, затем нажать **«Изменить пароль»**.

### 7.2 Управление двухфакторной аутентификацией

Для включения 2FA:

1. В разделе «Безопасность» нажать кнопку **«Включить 2FA»**.
2. Отсканировать отображённый QR-код с помощью приложения-аутентификатора.
3. Ввести сгенерированный 6-значный код для подтверждения.
4. Нажать **«Подтвердить»**.

Для отключения 2FA необходимо ввести текущий код из приложения-аутентификатора и нажать кнопку **«Отключить 2FA»**.

### 7.3 Загрузка аватара

В разделе профиля нажать кнопку **«Загрузить аватар»** и выбрать изображение в формате JPEG или PNG. Максимальный допустимый размер файла составляет 5 МБ.

### 7.4 Управление API-токенами

API-токены предназначены для программного доступа к REST API системы без использования сессионной аутентификации.

Для создания токена:
1. В разделе «API-токены» нажать кнопку **«Создать токен»**.
2. Присвоить токену наименование.
3. Скопировать и сохранить отображённое значение токена. Значение отображается только один раз и в дальнейшем недоступно.

Для отзыва токена нажать кнопку **«Удалить»** напротив соответствующей записи.

Токен передаётся в HTTP-заголовке:
```
Authorization: Bearer <значение_токена>
```

## 8. Управление словарями пейлоадов

### 8.1 Загрузка словаря

Для загрузки пользовательского словаря:

1. Перейти в раздел **«Словари»** через навигационное меню.
2. Нажать кнопку **«Загрузить словарь»**.
3. Выбрать текстовый файл (расширение `.txt`).

Требования к формату файла:
- каждый пейлоад располагается на отдельной строке;
- строки, начинающиеся с символа `#`, рассматриваются как комментарии и не используются при сканировании;
- кодировка файла: UTF-8;
- максимальный размер файла: 10 ГБ.

### 8.2 Использование словаря при сканировании

Все словари, загруженные пользователем, автоматически применяются при запуске сканирования. Дополнительно словари могут быть явно выбраны при создании задачи сканирования.

### 8.3 Удаление словаря

В разделе «Словари» нажать кнопку **«Удалить»** напротив соответствующей записи.

---

## 9. Административная панель

Раздел «Администратор» доступен только пользователям с ролью **admin**.

### 9.1 Управление пользователями

В подразделе **«Пользователи»** отображается полный список зарегистрированных учётных записей. Доступны следующие действия:

- **Деактивация** — блокировка учётной записи; пользователь теряет возможность входа в систему.
- **Активация** — восстановление доступа к ранее заблокированной учётной записи.

### 9.2 Управление API-токенами

В подразделе **«API-токены»** отображаются все токены, созданные пользователями системы. Администратор вправе отозвать любой токен, нажав кнопку **«Удалить»**.

### 9.3 Управление сессиями

В подразделе **«Сессии»** отображаются активные сессии всех пользователей с указанием IP-адреса, User-Agent и времени создания. Администратор вправе принудительно завершить любую сессию.
